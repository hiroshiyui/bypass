# bypass — browser extension

A small WebExtension that lets you search your `bypass` store
and copy passwords from the browser's toolbar. All crypto stays
in the desktop `bypass` binary; this extension is a thin UI
that talks to it over the browser's
[native messaging](https://developer.chrome.com/docs/extensions/develop/concepts/native-messaging)
channel.

- **Wire protocol**: [ADR-0022](../doc/adr/0022-native-messaging-wire-protocol.md)
- **Extension architecture**: [ADR-0023](../doc/adr/0023-browser-extension-architecture.md)

## Build

```sh
# From the repo root:
cd extension/
npm ci
node build.mjs           # bundles to dist/
node build.mjs --zip     # also writes bypass-extension-<version>.zip
```

Output goes into `extension/dist/` and (with `--zip`) a
`bypass-extension-X.Y.Z.zip` alongside.

## Install (development)

You need the `bypass` CLI installed and the native-messaging
manifest registered before either browser can talk to the host.

```sh
# Build the CLI
cargo install --path crates/bypass-cli

# Initialise your store (skip if migrating from `pass`)
bypass init you@example.com
bypass insert email/work    # add one entry so the popup has something to show

# Register the native host with both browsers.
# Chrome's extension id is autogenerated when you load unpacked
# (see step 3 of the Chrome flow below); re-run install after
# you have it.
bypass messaging-host install
```

### Firefox

1. Build the extension (`node build.mjs`).
2. Open `about:debugging` → "This Firefox" → "Load Temporary
   Add-on…" → pick `extension/dist/manifest.json`.
3. Click the toolbar icon. Search; click "Copy" to put a
   password on the clipboard.

The temporary add-on disappears when Firefox restarts.
Permanent install requires signing through AMO.

### Chrome / Chromium / Brave / Edge / Vivaldi

1. Build the extension (`node build.mjs`).
2. Open `chrome://extensions` (or the equivalent), enable
   "Developer mode", and choose "Load unpacked" → pick
   `extension/dist/`.
3. Copy the extension ID shown on the extensions page.
4. Re-register the native host with that ID so Chrome will
   actually launch it:
   ```sh
   bypass messaging-host install --chrome-id <id>
   ```
5. Click the toolbar icon. Search; click "Copy".

## Uninstall the native host

```sh
bypass messaging-host uninstall
```

Removes every per-browser manifest. The extension itself
uninstalls through the browser's extension page.

## Troubleshooting

- **"Cannot reach bypass host"** in the popup → the
  native-messaging manifest is missing or the binary path
  baked into it doesn't exist. Re-run
  `bypass messaging-host install` (with `--chrome-id` on
  Chrome) and check the path with
  `cat ~/.mozilla/native-messaging-hosts/io.bypass.host.json`.
- **"Specified native messaging host not found"** in Firefox →
  the gecko ID in `manifest.json` doesn't match the
  `allowed_extensions` array in the host manifest. Both must
  be `bypass@bypass.example` unless you've overridden via
  `bypass messaging-host install --firefox-id`.
- **gpg-agent prompts every time** → that's expected the first
  time per agent session. Subsequent decrypts use the cached
  passphrase for as long as `default-cache-ttl` allows. The
  CLI shares this behaviour; the extension inherits it.
- **Popup says "reply too large"** → the entry is bigger than
  ADR-0022's 512 KB cap. Fall back to `bypass show` in the
  CLI for that entry.

## What's not in v1

- **Autofill** (content scripts, password-field detection,
  click-to-fill). Phase 7.2.b.
- **Per-entry auto-clear** of the clipboard from the
  extension itself. The CLI's `bypass show -c` handles this
  via the existing clipboard daemon; the popup wires up the
  same path in a later iteration.
- **AMO / Chrome Web Store automation.** `build.mjs --zip`
  produces a submittable artefact; the actual upload is
  manual for now.
- **Icons.** The manifest currently has no `icons` field, so
  both browsers show a default puzzle-piece. Replace before
  store submission.
